Tutoriel OpenRefine

En ces temps de confinement intensif, la question que se posent tous ceux qui ne savent plus quoi faire de leurs dix doigts est : « Comment profiter de ce temps libre pour apprendre de nouvelles choses ». La démarche est louable. Si vous êtes ici c’est donc votre attirance pour l’univers de la datavisualisation qu’il faut remercier. J’espère que jusqu’à maintenant, la visite de cette vitrine modeste de mes travaux vous aura enchanté. J’ai donc décidé de vous faire profiter de mes humbles connaissances en la matière.

De quoi va-t-on parler ?

L’objectif à la fin du défilement de cette page web est de vous introduire à l’univers de la réconciliation et du nettoyage de jeux de données. Cette étape, méconnue et sous-estimée, est pourtant un atout précieux. Cela permettra de sélectionner les items les plus pertinent à utiliser dans votre dataviz mais aussi d’améliorer la sémantique de certaines de vos données en les connectant à d’autres bases de données. La finalité sera de produire une datavisualisation complète, riche d’informations et agréable à l’œil.

J'ai besoin de quoi ?

  • Un ordinateur connecté à internet (oui, lâchez votre smartphone)
  • Le logiciel OpenRefine disponible ici
  • Le site web RawGraph
  • Un jeu de données au format .csv (à trouver sur data.gouv, kaggle, google dataset search)
  • Un peu de concentration (j’ai dit un peu)

Réconciliation et nettoyage

Durant l’ensemble de ce tutoriel j’utiliserai le jeu de données suivant que j’ai nettoyé au préalable afin d’aller à l’essentiel : ici. Il propose la liste des films Marvel estampillés « Marvel Cinematic Universe ». On retrouve pour chaque film son nom, ses réalisateurs et producteurs, le coût et la recette, la durée, un lien vers la page imdb.com, etc. Mon conseil pour débuter : Utiliser un jeu de données léger et sans trop de lignes afin qu’OpenRefine puisse traiter vos actions dans les meilleurs délais. Commencez par importer votre fichier dans le logiciel

Vérifier l’encodage du texte pour éviter de mauvaise surprise. Sélectionnez UTF-8.

Pour plus de clarté par la suite veillez à renommer votre projet

Comme dit plus haut, l’intérêt d’OpenRefine est de lier des éléments de votre jeu de données à d’autres bases de données. Ici nous utiliserons l’extension Wikidata. Cliquez sur la flèche dans la case title. Déroulez le menu jusqu’à pouvoir sélectionner « start reconciling »

Ici vous pourrez choisir votre base de données. Vous comprenez donc que vous pouvez en ajouter. Pour l’instant, si vous n’avez jamais utilisé le logiciel il n’y a que Wikidata. Automatiquement, OpenRefine vous propose différentes entités. Sélectionnez la plus pertinente. Ici « film ». Cliquez sur « Start Reconciling »

Un chargement se lance. Plus le jeu de données sera grand, plus cette étape sera longue. Deux cas se présentent maintenant. Dans le premier, le logiciel a bien travaillé et le titre du film est relié à l’entité wikidata. Dans un second, le logiciel vous propose différents possibilité. A droite on trouve un score de pertinence. Vous avez juste à cliquer sur « match this cell »

Une fois cela fait, c’est là que tout devient plus intéressant. Désormais vous pouvez aller chercher de nouvelles propriétés sur wikidata. Pour cela, faites de nouveau apparaitre le menu déroulant pour finalement accéder à « Add columns from reconciled values… »

Automatiquement, on vous propose d’ajouter une colonne. Choisissez ainsi les données que vous voulez ajouter. J’ai décidé d’ajouter les réalisateurs.

Si jamais ne vous faites une erreur, utilisez la fonction undo/redo qui vous permet de remonter en arrière dans vos modifications.

Pour exporter votre projet, cliquez sur export en haut à droite. Choisissez ensuite le type de format.
